# Yashveer Labs · Operating System

The Yashveer Labs website rebuilt as a multi-page, dock-navigated experience. Next.js 15 App Router, React 19, TypeScript strict, Tailwind v4 with a CSS variable color system, Framer Motion + GSAP + Lenis for choreography, React Three Fiber for 3D, and a JSON-LD foundation for SEO, GEO, and AEO discoverability.

## Stack

- Next.js 15 (App Router) on React 19, TypeScript strict.
- Tailwind v4 with inline `@theme`, color system in CSS variables.
- Framer Motion (dock physics, folder morph, page transitions).
- GSAP (heavier scroll choreography when needed).
- Lenis (smooth scroll, respects `prefers-reduced-motion`).
- React Three Fiber + Drei + Postprocessing (hero sigil, Nyxera core, ambient backdrops).
- Resend + Zod for the contact form.
- next-sitemap for `sitemap.xml` and `robots.txt`.
- schema-dts for typed JSON-LD.

## Local development

```bash
cd Public
npm install
cp .env.example .env.local   # then fill in RESEND_API_KEY when ready
npm run dev
```

Open `http://localhost:3000`.

## Build

```bash
npm run build
```

`postbuild` runs `next-sitemap` and writes `public/sitemap.xml`, `public/sitemap-0.xml`, `public/robots.txt`.

## Environment variables

| Key | Required | Purpose |
| --- | --- | --- |
| `RESEND_API_KEY` | For real email delivery | Used by `/api/contact`. Without it, the form logs the payload and returns 200 (dev-friendly). |
| `CONTACT_TO_EMAIL` | Optional | Defaults to `yashveersr4@gmail.com`. |
| `CONTACT_FROM_EMAIL` | Optional | Defaults to `Yashveer Labs <onboarding@resend.dev>`. Set a verified address from your Resend dashboard for production. |
| `SITE_URL` | Optional | Defaults to `https://yashveerlabs.vercel.app`. |

## Deploy to Vercel

1. Push this folder to GitHub (the `Public/` directory is the project root).
2. In Vercel, import the repo. Set the root directory to `Public` if Vercel does not auto-detect.
3. Set the environment variables above in **Project Settings → Environment Variables**.
4. Deploy. Subsequent pushes auto-deploy.

## Routes

Direct dock pages: `/`, `/projects`, `/about`, `/resume`, `/blog`, `/contact`, `/nyxera`.

Folder pages (reached via dock folder morph):

- **Social** (external links): Instagram, LinkedIn, GitHub, YouTube, Pinterest, X.
- **Why Not → Systems**: `/systems/stack`, `/systems/tools`, `/systems/technologies`, `/systems/systems`, `/systems/engineering-capabilities`.
- **Why Not → Journey**: `/journey`, `/journey/achievements`, `/journey/milestones`, `/journey/work-history`, `/journey/evolution`.
- **Why Not → Labs**: `/labs/ai-experiments`, `/labs/robotics`, `/labs/vision-systems`, `/labs/prototypes`, `/labs/interfaces`, `/labs/concepts`.

Blog posts live under `/blog/[slug]` and are statically generated from `lib/content/blog.ts`.

## Design system

The full color system lives as CSS variables in `app/globals.css`. The rule:

- Dark backgrounds 85 percent of the UI.
- Neutral text / borders / chips 10 percent.
- Champagne gold accent ≤ 5 percent.
- Nyxera cyan only on the Nyxera surface and the Nyxera dock icon.
- No pure black, no em-dashes, no neon, no large gold surfaces.

## Content authoring notes

The voice on every page is first person, direct, short rhythmic sentences, no AI-tells, no em-dashes. The Why Me cards and project case studies live in `components/sections/WhyMe.tsx` and `lib/content/projects.ts` respectively. Anything still pending real input (the `/about` portrait drop-in being the main one) is intentionally left so the layout renders gracefully without it.

## Performance budget

Lighthouse targets: Performance 85+, Accessibility 95+, Best Practices 95+, SEO 100. 3D scenes lazy-load with `ssr: false`, cap DPR at 2, and pause when the tab is hidden. Mobile drops magnification and uses simpler scenes.
